# 등불 배포 가이드

## 0. 이번 업데이트 내용 (API 호출 최적화)
질문 1건당 Claude API 호출 횟수를 **2회 → 1회**로 줄였습니다.
- 기존: "관련 구절 후보 찾기" 호출 + "그중 3~5개 선택해서 조언 쓰기" 호출 (2회)
- 변경: 구절 선택과 조언 작성을 한 번의 호출에서 함께 처리 (1회), 예비 구절(backup) 2개를 같이 받아서
  혹시 성경 데이터에서 못 찾는 구절이 있어도 자동으로 대체됩니다.
- 로컬 표본(98구절) 대체 경로도 "분류 → 조언 작성" 2회에서 **1회**로 통합했습니다.

또한 **재시도(retry) 로직**을 추가했습니다.
- 서버(`api/claude.js`): Anthropic API가 429(요청 초과)나 5xx(일시 장애) 응답을 주면, 0.6초 → 1.2초
  간격으로 최대 2회 자동 재시도합니다.
- 프론트(`index.html`): 서버 응답이 429/502/503이면 클라이언트에서도 0.8초 → 1.6초 간격으로 최대 2회
  재시도하며, 화면에 "지금 이용자가 많아 잠시 대기 중입니다..." 안내를 보여줍니다.

이 두 가지 덕분에 트래픽이 몰려도 즉시 에러를 띄우기보다 자동으로 완충되고, 전체 API 사용량 자체도
절반 가까이 줄어 같은 사용량 한도로 더 많은 이용자를 받을 수 있습니다.

## 구조
```
deploy/
├── index.html          # 프론트엔드 (정적 파일)
├── bible_kr.json        # 성경 전체 텍스트 (66권, 약 4.4MB)
├── manifest.json         # PWA 설정 (앱 이름, 아이콘, 색상)
├── sw.js                 # 서비스 워커 (오프라인/캐싱)
├── icons/                # 홈 화면 아이콘 (다양한 크기)
├── api/
│   └── claude.js         # 서버리스 함수 — Claude API 키는 여기에만 존재
├── package.json
├── .env.example
└── .gitignore
```

프론트엔드는 더 이상 `api.anthropic.com`을 직접 호출하지 않습니다. 대신 `/api/claude`를 호출하고,
그 서버리스 함수가 실제 Anthropic API 키를 사용해 요청을 대신 처리합니다. 이렇게 해야 브라우저
개발자도구로 키가 노출되는 것을 막을 수 있습니다.

## 1. Vercel로 배포하기 (권장, 무료 티어로 충분)

### 방법 A — CLI (가장 빠름)
```bash
npm install -g vercel
cd deploy
vercel login
vercel
```
질문에 기본값으로 답하면 됩니다 (프레임워크: Other / 빌드 명령 없음).

### 방법 B — GitHub 연동
1. 이 `deploy` 폴더 내용을 GitHub 저장소에 올리기
2. https://vercel.com → New Project → 해당 저장소 선택 → Deploy

### API 키 등록 (필수)
Vercel 대시보드 → 프로젝트 → **Settings → Environment Variables**
- Key: `ANTHROPIC_API_KEY`
- Value: 실제 Anthropic API 키 (`console.anthropic.com`에서 발급)
- Environment: Production, Preview, Development 모두 체크

등록 후 **Redeploy**를 한 번 눌러야 반영됩니다.

## 2. 배포 후 확인
- 사이트 접속 → 고민 입력 → "빛 비추기" 클릭 → 정상적으로 구절과 조언이 뜨는지 확인
- 브라우저 개발자도구 Network 탭에서 `/api/claude` 요청은 보이지만, 그 안에 API 키가 없는지 확인
- 잘못된 방식으로 연속 요청 시 (`429` 응답) rate limit이 작동하는지 확인

## 2-1. PWA(홈 화면 앱)로 설치 확인하기
이 앱은 PWA로 만들어져 있어서, 배포된 주소로 스마트폰에서 접속하면 홈 화면에 앱처럼 설치할 수 있습니다.
- **안드로이드(크롬)**: 사이트 접속 시 하단에 "설치하기" 배너가 자동으로 뜹니다. 안 뜨면 메뉴(⋮) → "앱 설치"
- **아이폰(사파리)**: 사이트 접속 시 상단에 안내 배너가 뜹니다. 공유 버튼(□↑) → "홈 화면에 추가"
- 설치 후 아이콘을 누르면 주소창 없이 전체화면 앱처럼 열립니다
- `sw.js`(서비스 워커)가 `index.html`, `bible_kr.json`, 아이콘을 캐싱해서 재방문 시 더 빠르게 열리고, 오프라인에서도 화면은 뜹니다 (단, 조언 생성은 인터넷 필요)
- 코드를 수정한 뒤 재배포했는데 앱에 옛날 화면이 계속 보이면: `sw.js`의 `CACHE_VERSION` 값을 `deungbul-v2`처럼 바꿔서 재배포하세요 (캐시 강제 갱신)

## 3. 배포 전 반드시 확인할 것

### 저작권 (중요)
업로드해주신 성경 본문은 표현상 **개역개정(1998)**으로 보이며, 대한성서공회가 저작권을
보유한 번역본입니다. 개인적 테스트/비공개 용도는 문제없지만, **일반에 공개하는 서비스로
배포하려면 대한성서공회에 사용 허가를 받아야** 합니다.
- 안내: https://www.bskorea.or.kr (성서에 관하여 > 성경의 저작권)
- 대안: 저작권이 만료된 개역한글판(1961, 2012년 저작권 만료)으로 교체하면 이 절차가 필요 없습니다.

### 남용 방지
`api/claude.js`에 기본적인 IP 기반 rate limit(분당 15회)을 넣어뒀지만, 이는 서버리스 함수의
메모리에 의존하는 **best-effort 수준**입니다. 트래픽이 늘어나면 다음을 고려하세요:
- Upstash Redis 등 외부 저장소 기반 rate limit으로 교체
- Vercel의 Firewall / Attack Challenge Mode 활성화 (대시보드에서 설정 가능)
- Anthropic 콘솔에서 사용량 알림(budget alert) 설정

### 위기 상황 대응
자살·자해 등 위기 신호가 있는 대화에서는 앱이 전문기관 안내를 함께 제공하도록 프롬프트에
넣어뒀지만, 실제 서비스라면 화면에 상시 노출되는 위기 상담 연락처(예: 자살예방상담전화 1393)도
함께 배치하는 것을 권장합니다.

## 4. 다른 호스팅을 쓰고 싶다면
- **Netlify**: `api/claude.js`를 `netlify/functions/claude.js`로 옮기고 `exports.handler`
  형태로 살짝 문법만 바꾸면 동일하게 동작합니다.
- **Cloudflare Pages**: Pages Functions(`functions/api/claude.js`)로 이식 가능, 문법이
  약간 다릅니다 (`onRequestPost` export 방식).

구조 자체(정적 프론트 + API 키를 숨기는 서버리스 함수)는 어떤 호스팅에서도 동일하게 적용됩니다.
